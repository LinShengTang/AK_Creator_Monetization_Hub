Recommended Learning Order

建議學習順序｜先主線，再補強

這個順序是「內容作戰主線」，不是工具執行流程。先決定方向，再選題、寫開場、完成腳本、承接轉化，最後用數據與執行節奏回收。

01

Strategy｜定位 / 受眾 / 內容主線 / 轉化路徑

先決定你服務誰、用什麼定位切入、有哪些內容主線，以及最終導向哪條低壓力變現路徑。主要用在所有內容開始前的方向裁決。

 02

Signal｜題材判斷 / 選題系統

判斷什麼題材值得做，什麼只是噪音。主要用在把留言、搜尋、私訊、競品與後台數據轉成可執行選題。

 03

Hook｜第一句 / 開場截停 / 承諾設計

讓受眾在前 1–3 秒停下來，但不使用誇大承諾、假爆款或標題黨。

 04

Script｜腳本承諾兌現 / 留存結構

把 Hook 的承諾轉成可看、可懂、可拍、可生成的內容結構。

 05

CTA｜低壓行動 / 安全互動

讓觀眾知道下一步，但避免留言+1、按讚解鎖、假稀缺或 engagement bait。

 06

Stories｜信任加溫 / DM 前置橋接

把觀看後的短暫注意力延伸成信任、回覆與對話起點。

 07

Conversion｜轉換路徑 / 低壓成交 / Offer 承接

設計觀看後的下一步：DM、表單、低壓諮詢、產品或服務。這是主線 Conversion，不是 DM 卡關補強模組。

 08

Algorithm｜平台規則 / 指標邊界

理解 YouTube / Instagram / Facebook 的推薦訊號、資料邊界與不可誇大的演算法說法。

 09

Performance｜數據診斷 / 下一刀修正

發布後回收數據，判斷下一支影片應優先修正哪一個斷點；這張卡固定連到 Performance Manual。

 10

Execution｜可持續執行 / 工作流節奏

把內容流程變成可持續週期，而不是只靠靈感或一次性爆發。

Reinforcement Modules

補強模組｜四大漏斗斷點

這些不是新的主線章節，也不是每天都要照順序讀。它們是「斷點修復工具」：主線流程先跑，發現漏點時才回頭使用對應補強模組。

什麼時候使用補強模組？

主頁瀏覽高，但私訊 / 連結點擊低
使用：主頁防漏
DM 已讀不回、嫌貴、推託變多
使用：DM 非線性決策樹
有免費 / 低價入口，但無法升級後端服務
使用：Value Ladder
Hook 文案不差，但前 0.5 秒滑走率高
使用：Visual Pattern Interrupt

主頁防漏｜個人簡介 / 置頂內容 / 精選限動

補上短影片或貼文之後，受眾點進主頁時的二次審查與承接層，避免冷流量在 Profile 流失。

DM 非線性決策樹｜已讀不回 / 嫌貴 / 推託應對

補強 Conversion v1.0：當 DM 對話沒有照標準五步流程前進時，用安全、低壓的方式判斷下一步。

Value Ladder｜後端價值梯隊 / 升級路徑

補上免費資源、低價入口、中價解法、高價值後端之間的安全升級路徑與第 0 / 3 / 7 / 14 天追蹤協議。

Visual Pattern Interrupt｜前 0.5 秒視覺截停

補強 Hook：把文字開場轉成第一幀畫面、第一個動作與視覺中斷，降低觀眾還沒讀到 Hook 就滑走的機率。
